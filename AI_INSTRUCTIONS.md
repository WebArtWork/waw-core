waw-core is the foundational system module of the waw platform that provides the default CLI command set and a small, stable runtime extension to the shared `waw` execution context. It is responsible for project scaffolding via `waw new`, module scaffolding via `waw add` or `waw a`, CSS framework switching for supported front-end project types via `waw css`, module repository synchronization and publishing via `waw sync`, version reporting via `waw version` and related aliases, PM2-based process management via `waw start`, `waw stop`, and `waw restart`, and a small set of runtime helpers attached during startup. When loaded at runtime, waw-core extends `waw` with `waw.wait(ms)`, a Promise-based delay helper, and `waw.http(hostname, port)`, a minimal HTTP client wrapper supporting standard HTTP verbs with configurable headers and JSON handling. CLI commands are exposed through `server/core/cli.js` and delegate implementation to focused utility files, while runtime behavior is initialized through `server/core/index.js`. waw-core intentionally does not implement application frameworks, routing, databases, or business logic, and exists solely to provide stable developer workflows and shared low-level utilities that other modules can rely on.
