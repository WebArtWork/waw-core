waw-core is the foundational system module of the waw framework that provides the **standard CLI command set** and a **small, stable runtime extension** to the shared `waw` context. It provides project scaffolding via `waw new`, module scaffolding via `waw add` / `waw a`, CSS framework switching via `waw css` for supported front-end project types, module repository synchronization via `waw sync` (both force-sync and publish modes), Git command workflows via `waw git`, and PM2 process management via `waw start`, `waw stop`, and `waw restart`. At runtime, it augments `waw` with `waw.wait(ms)` (Promise-based delay) and `waw.http(hostname, port)` (minimal HTTP helper with `get/post/put/patch/delete` and configurable `headers`). Core also standardizes interactive CLI UX by using the framework-provided `waw.terminal()` utility: create an instance with `const t = waw.terminal();`, then use `await t.ask(question, opts)` for text input, `await t.confirm(question, opts)` for yes/no prompts, and `await t.choose(title, options, opts)` for menu selection (options can be strings or `{ label, value, detail, disabled }`), optionally using `t.spinnerStart()/t.spinnerUpdate()/t.spinnerStop()` during long operations, and finishing with `t.close()` to cleanly close the session.
